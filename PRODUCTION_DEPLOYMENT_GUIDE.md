## ✅ Pre-Deployment Checklist

### 1. Code Quality & Security

- [ ] All environment variables documented in `.env.example`
- [ ] No secrets or credentials committed to Git
- [ ] Database schema is up to date with migrations
- [ ] TypeScript compilation passes with no errors
- [ ] All tests pass (if applicable)

### 2. Environment Variables Prepared

Gather these values before deployment:

**Required:**
- `JWT_SECRET` - Will be auto-generated by Render
- `FRONTEND_URL` - Will be auto-set from Render services
- `SMTP_HOST` - Your email provider's SMTP host (e.g., smtp.gmail.com)
- `SMTP_PORT` - Usually 587 for TLS
- `SMTP_USER` - Your email address
- `SMTP_PASSWORD` - Your email password or app-specific password

**Optional (for avatar uploads):**
- `R2_ACCOUNT_ID` - Cloudflare R2 account ID
- `R2_ACCESS_KEY_ID` - R2 API access key
- `R2_SECRET_ACCESS_KEY` - R2 API secret key
- `R2_BUCKET_NAME` - R2 bucket name (default: contribly-db)
- `R2_PUBLIC_URL` - R2 public URL (e.g., https://pub-xxx.r2.dev)

**Optional (for Google OAuth):**
- `GOOGLE_CLIENT_ID` - Google OAuth client ID
- `GOOGLE_CLIENT_SECRET` - Google OAuth client secret
- `GOOGLE_CALLBACK_URL` - Will be set to your API URL + /api/auth/google/callback

### 3. Git Repository

- [ ] Code pushed to GitHub/GitLab/Bitbucket
- [ ] Repository is accessible (public or connected to Render)
- [ ] `render.yaml` is in the root directory

## 🚀 Deploy to Render

### Method 1: Using render.yaml (Recommended)

1. **Push your code to Git:**
   ```bash
   git add .
   git commit -m "Prepare for production deployment"
   git push origin main
   ```

2. **Go to Render Dashboard:**
   - Navigate to https://dashboard.render.com
   - Click **"New"** → **"Blueprint"**

3. **Connect Repository:**
   - Select your Git provider
   - Choose the `contribly` repository
   - Render will detect `render.yaml`

4. **Review Services:**
   - You should see 3 services:
     - `contribly-db` (PostgreSQL)
     - `contribly-api` (Express API)
     - `contribly-web` (Next.js frontend)
   - Click **"Apply"**

5. **Wait for Deployment:**
   - Database will be created first (~2-3 minutes)
   - Then API will build and deploy (~5-10 minutes)
   - Finally frontend will deploy (~5-10 minutes)

### Method 2: Manual Setup

If you prefer not to use Blueprint:

1. **Create PostgreSQL Database:**
   - New → PostgreSQL
   - Name: `contribly-db`
   - Region: Oregon (or closest to your users)
   - Plan: Free (or Standard for production)

2. **Create API Service:**
   - New → Web Service
   - Connect repository
   - Name: `contribly-api`
   - Environment: Node
   - Build Command: (see Build Commands section below)
   - Start Command: `cd apps/api && npm start`
   - Add environment variables (see Environment Variables section)

3. **Create Frontend Service:**
   - New → Web Service
   - Connect repository (same repo)
   - Name: `contribly-web`
   - Environment: Node
   - Build Command: (see Build Commands section below)
   - Start Command: `cd apps/web && npm start`
   - Add environment variables

## 🔧 Post-Deployment Configuration

### 1. Add SMTP Credentials

In Render dashboard:

1. Go to `contribly-api` service
2. Click **"Environment"** tab
3. Add/Update these variables:
   - `SMTP_HOST` = your SMTP host
   - `SMTP_USER` = your email
   - `SMTP_PASSWORD` = your email password
4. Click **"Save Changes"**
5. Service will automatically redeploy

### 2. Add R2 Credentials (Optional)

If using Cloudflare R2 for avatar storage:

1. In `contribly-api` environment:
   - `R2_ACCOUNT_ID`
   - `R2_ACCESS_KEY_ID`
   - `R2_SECRET_ACCESS_KEY`
   - `R2_PUBLIC_URL`
2. Save and redeploy

### 3. Add Google OAuth (Optional)

If using Google sign-in:

1. In `contribly-api` environment:
   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`
   - `GOOGLE_CALLBACK_URL` = `https://your-api-url.onrender.com/api/auth/google/callback`
2. Save and redeploy

### 4. Update CORS (if using custom domain)

If you have a custom domain:

1. In `contribly-api` environment:
   - Add `CORS_ORIGIN` = `https://yourdomain.com`
2. Save and redeploy

## ✅ Verification

### Automated Verification

Run the deployment verification script:

```bash
npx tsx scripts/verify-deployment.ts https://contribly-api.onrender.com
```

This will check:
- ✅ Health check endpoint responds
- ✅ CORS headers are configured
- ✅ Login endpoint exists
- ✅ Registration endpoint exists

Expected output:
```
✅ PASS   | Health Check
✅ PASS   | CORS Headers
✅ PASS   | Login Endpoint
✅ PASS   | Registration Endpoint

🎉 All critical checks passed! Deployment looks healthy.
```

### Manual Verification

1. **Check API Health:**
   ```bash
   curl https://contribly-api.onrender.com/health
   ```
   Should return:
   ```json
   {
     "status": "ok",
     "timestamp": "2026-02-23T...",
     "version": "0.1.0"
   }
   ```

2. **Check Frontend:**
   - Visit `https://contribly-web.onrender.com`
   - Should see the landing page
   - Try registering a new account
   - Try logging in

3. **Check Logs:**
   - In Render dashboard, go to each service
   - Click **"Logs"** tab
   - Look for startup messages:
     ```
     ✅ All required environment variables are present
     🚀 API server running at http://localhost:3001
     ✓ All routes registered successfully
     ```

## 🐛 Troubleshooting

### Issue: API Won't Start

**Symptom:** Logs show: "Missing required environment variables"

**Solution:** Add missing variables in Render dashboard → Environment tab

---

### Issue: Database Connection Failed

**Symptom:** Logs show: "Can't reach database server"

**Solution:**
1. Check `DATABASE_URL` is set correctly
2. Ensure database is in same region as API
3. Check database status (should be "Available")

---

### Issue: CORS Errors in Browser

**Symptom:** Browser console shows: "blocked by CORS policy"

**Solution:**
1. Check `FRONTEND_URL` matches your actual frontend URL
2. Check API logs to see which origin was rejected
3. Add origin to `CORS_ORIGIN` if using custom domain

---

### Issue: 404 on API Endpoints

**Symptom:** `/api/auth/login` returns 404

**Solution:**
1. Check API logs for "Failed to load routes"
2. Ensure Prisma client was generated during build
3. Check build logs for errors
4. Manually redeploy API service

---

### Issue: Slow First Request (Free Tier)

**Symptom:** First request takes 20-30 seconds

**Explanation:** Free tier services "sleep" after inactivity

**Solutions:**
- Upgrade to Starter plan ($7/month) for always-on
- Use a service like UptimeRobot to ping every 10 minutes
- Accept the delay (only affects free tier)

## 📊 Monitoring & Maintenance

### Health Monitoring

Set up external monitoring (recommended):

1. **UptimeRobot** (free):
   - Add HTTP monitor
   - URL: `https://contribly-api.onrender.com/health`
   - Check every 5 minutes
   - Alert on 3 failures

2. **Render Built-in:**
   - Automatically checks `/health` endpoint
   - Auto-restarts if 3 consecutive failures
   - View in Render dashboard → Metrics

### Database Backups

**Free Tier:**
- 7-day rolling backups (automatic)
- Download: Render dashboard → Database → Backups

**Paid Tiers:**
- 30-day rolling backups
- Point-in-time recovery

### Logs

**View Logs:**
```bash
# Using Render CLI (if installed)
render logs --service contribly-api --tail

# Or in dashboard:
# Render Dashboard → contribly-api → Logs
```

**Log Prefixes:**
- `✅` = Success/Info
- `❌` = Error
- `⚠️` = Warning
- `🔒` = Production CORS enabled
- `🚀` = Server started

### Performance Monitoring

**Free Tier:**
- Basic metrics in Render dashboard
- Response time graphs
- Memory usage

**Recommended (Paid):**
- Sentry for error tracking
- Datadog/New Relic for APM
- LogDNA/Papertrail for log aggregation

## 🔄 Updates & Migrations

### Deploying Code Updates

1. Push to Git:
   ```bash
   git add .
   git commit -m "Your update message"
   git push origin main
   ```

2. Render auto-deploys on push (if enabled)
   - Or manually: Dashboard → Service → "Manual Deploy"

3. Database migrations run automatically during API build

### Rolling Back

1. Go to Render dashboard → Service
2. Click **"Deploys"** tab
3. Find previous successful deploy
4. Click **"Redeploy"**

### Database Migrations

**Forward (Automatic):**
- Migrations run during API build
- Command in `render.yaml`: `npx prisma migrate deploy`

**Rollback (Manual):**
```bash
# SSH into Render shell (Paid tier only)
# Or run locally then push:
npx prisma migrate resolve --rolled-back migration_name
```

## 🎯 Production Best Practices

### Security

- [ ] Use HTTPS only (Render provides free SSL)
- [ ] Rotate JWT_SECRET periodically
- [ ] Use environment-specific secrets (never commit)
- [ ] Enable 2FA on Render account
- [ ] Review database access logs

### Performance

- [ ] Use Starter plan or higher for no sleep
- [ ] Enable HTTP/2 (automatic on Render)
- [ ] Monitor response times in dashboard
- [ ] Consider CDN for static assets (Cloudflare)

### Cost Optimization

**Free Tier ($0/month):**
- 750 hours/month free
- Sleeps after 15 min inactivity
- Good for: Testing, personal projects

**Starter Plan ($7/service/month):**
- Always on, no sleep
- 400GB bandwidth
- Good for: Small production apps

**Standard Plan ($25/service/month):**
- Horizontal scaling
- More memory/CPU
- Priority support
- Good for: Growing apps with traffic
